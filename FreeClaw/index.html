Features

Everything you need.
Nothing you don't.

FreeClaw classifies your intent locally before touching any API, then decides how much history to send, how cold to run, and which tools to offer — per message.

🌐
Web Search & Scraping
Pulls instant answers, news, and snippets from DuckDuckGo, then scrapes and cleans the top result pages — all stitched into one structured block. No extra model call spent summarizing.
🧠
Persistent Memory
Durable facts about you live in context.md, read and updated with the same file tools the agent uses for everything else — without that history bloating the active context window every turn.
🖥️
Bash Execution
Runs shell commands directly on your machine and returns the output. Full access, no sandboxing.
🧩
MCP Servers
Connect external Model Context Protocol servers from Settings → MCP Servers. Their tools are fetched over Streamable HTTP and merged into the agent's toolset automatically — no restart required.
📄
Files, Pages & Vision
Creates, reads, edits, deletes, and lists files in its sandboxed static folder; publishes live HTML pages at a public URL; and describes uploaded images in detail using a vision model.
⚡
Adaptive Turns
Classy-AI tags your message locally before any API call, and the tag sets the turn: how much history goes out, the sampling temperature, and which tools are offered. Small talk gets a tiny window and no tools; precision work runs colder with everything.
💬
Minimal Context Windows
Only the most relevant recent messages go to the API per turn. Keeps token usage near zero for everyday use.
🔒
Password-Protected UI
The web chat sits behind a login screen, so it's safe to leave running and reachable on your local network.
🔀
Multi-Provider Fallback
Add any OpenAI-compatible endpoint from Settings → Providers — URL, key, model. The agent tries them in the order you list them and falls through to the next if one errors out or rate-limits you. No keys are collected at install.
👥
Multi-User Support
Each user gets their own folder, conversation history, and long-term memory. The agent can create new users on request with the create_user tool.
🖥️
Terminal CLI
A full-featured command-line interface alongside the web UI — same agent, same tools, rendered with color and live streaming directly in your terminal.
🔌
OpenAI-Compatible API
Exposes a /v1/chat/completions endpoint so any OpenAI SDK client can talk to FreeClaw directly. Auth uses your FreeClaw password as the Bearer token.
How It Works

From your browser
to the model.

Every message passes through a cost-optimization layer before a single token reaches your provider.

01
You sign in and type in the web UI
FreeClaw runs a local Flask server on port 6767, behind a password login. Open it in any browser on your network, sign in, type your message and hit Enter.
02
Local intent classification
Before any API call, Classy-AI reads your message locally and tags the intent — greeting, banter, search, logic, math, coding, editing — then picks the context window, temperature, and toolset for that turn. Zero tokens spent.
03
Provider call with trimmed context
Only the relevant recent messages — 3 to 9 of them — go out to the first enabled provider in Settings → Providers. If it errors out or rate-limits, FreeClaw falls through to the next provider in your list automatically.
04
Tool execution (if needed)
If the model calls a tool — web search (capped at 2 calls per task), bash, file and page management, image description, opening a URL, or anything exposed by a connected MCP server — FreeClaw executes it locally and feeds the result back in a recursive call.
05
Response rendered in the UI
The reply is streamed back to your browser as server-sent events and rendered with full markdown — code blocks, lists, headers, links. A live token counter tracks usage.

Token Savings

Engineered to cost
almost nothing.

FreeClaw stacks multiple strategies to minimize every API call. The result is a capable agent that runs all day for fractions of a cent.

Strategy 01
No tools for small talk
greeting · banter · about-user

Conversational intents are sent with no tool definitions at all and a 3–5 message window. Tool schemas are only paid for when the task actually calls for them.

Strategy 02
Trimmed context windows
3–9 recent messages sent per turn (vs full history)

Classy-AI sets the context window size per intent. A greeting sends 3 recent messages. Coding or editing sends up to 9. Full history is never blindly sent.

Strategy 03
Capped search calls
max 2 searches per task

The agent is hard-limited to 2 web search calls per task. If sufficient data isn't found, it reports back rather than spiraling into extra API calls.

Strategy 04
Zero-token classification
100% local · 0 tokens

Intent classification runs entirely on your machine using a local PyTorch model. Routing decisions cost nothing — no API call, no tokens, no latency.

Strategy 05
Memory, not replay
write once · context.md

Durable facts about you are written once to context.md instead of being re-sent in every conversation's context window.

Strategy 06
No LLM call for search
DuckDuckGo + BeautifulSoup

Search results are gathered, filtered for freshness, and structured entirely with deterministic code — not a separate model call.

Commands

Type these in the chat.

Special commands are handled directly in the chat box — no separate terminal needed.

Command What it does
/reset Clears the conversation history and starts fresh
/startapi Enables the OpenAI-compatible API at /v1/chat/completions (works in web UI and CLI)
/stopapi Disables the API (works in web UI and CLI)
/exit Exits the terminal CLI
Agent Tools

These are the tools the agent can call on your behalf.

Tool What it does
search Searches the web and returns results (max 2 calls per task)
read_web Reads the first 3000 characters of a specific URL
create_file Writes a new file to the /static directory and returns a public URL
edit_file Edits an existing file by replacing a specific string with a new one
delete_file Deletes a file from the /static directory
read_file Reads the contents of a file in the /static directory
list_files Lists all files in the /static directory
create_page Creates a live HTML page and returns a public URL
get_image_description Returns a detailed description of an uploaded image using a vision model
create_user Creates a new FreeClaw user with their own folder, conversation history, and long-term memory
open_url Opens a URL or custom URI on your device — a webpage, or an app like your texting or phone client
run_bash_command Executes a bash command on your machine and returns the output

Long-term memory isn't a separate tool — context.md is just a file in the agent's static folder, read and edited with read_file and edit_file like anything else. Any tool exposed by a connected MCP server is merged into this list automatically.

Installation

One command.
You're done.

The installer handles everything. You just need git, python3, and sudo. Linux only.

terminal
$ curl -fsSL https://freeclaw.eedeb.dev/install.sh | bash
🐧 Linux only
1
Clone & set up environment
The script clones the repo, creates a Python virtual environment, and installs all dependencies including PyTorch (CPU), Flask, the OpenAI SDK, and the Classy classifier.
2
Set a password for the web UI
FreeClaw's chat UI sits behind a login screen. You'll set a password during install so it's safe to leave running on your network. That's the only thing collected here — no API keys. It's written to a local .env file, never committed to the repo.
3
FreeClaw starts automatically
The installer registers FreeClaw as a systemd service so it starts on boot, installs the freeclaw terminal client, and prints the local URL — something like http://192.168.x.x:6767.
4
Add a provider in the web UI
Open the URL, sign in, and head to Settings → Providers. Add any OpenAI-compatible endpoint — URL, API key, model name — and stack as many as you like for fallback. FreeClaw can't answer until at least one is configured. Optionally add MCP servers under Settings → MCP Servers while you're there.
5
Start chatting
Chat from any browser on your network, or run freeclaw for the same agent in your terminal. Later on, ./update.sh from the install directory pulls the latest changes and restarts the service, leaving your data alone.
